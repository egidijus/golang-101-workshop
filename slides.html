$GOPATH

Why should I care?

The GOPATH environment variable is used to specify directories outside of $GOROOT that contain the source for Go projects and their binaries.

Package Management

something like pip install ??

the golang standard for packages is "go get", everything is always pulled from master github.com. golang was designed for monorepos, there is an expectation from the maintainer to never break compatibility. This might not work for everyone, because programming is hard, maintaining projects is even harder.

workspaces and go 1.11 (go mod)

do not fear, go mod is here

with go mod you can pin packages and have multiple workspaces. as in, each project can have it's own workspace and it's own dependencies, just like python requirements.txt .
